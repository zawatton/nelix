#+TITLE: Doc 23 — Nelix distro publication workflow

* Goal

Nelix should be installable first from the host distribution package
manager, then use its manifest and profile machinery to manage Emacs
packages and Linux user tools consistently across distributions.

Initial target:

- Debian / Ubuntu via APT
- Fedora via dnf

Rootless local pre-publication gate:

#+begin_src shell
make publication-local-gate
#+end_src

This combines =make apt-http-gate= with =make fedora-source-gate=.
It proves the signed/static APT repository can be served and consumed
locally, and that the Fedora source tarball is generated without local
build debris.  Fedora RPM install smoke remains covered by
=make fedora-container-gate= or a persistent Fedora host.

The local APT repository target creates:

#+begin_example
../nelix-apt-repo/
  pool/main/n/nelix/elpa-nelix_0.1.0-4_all.deb
  dists/unstable/Release
  dists/unstable/InRelease
  dists/unstable/Release.gpg
  dists/unstable/main/binary-all/Packages
  dists/unstable/main/binary-all/Packages.gz
  nelix-archive-keyring.gpg
  sources.list
  sources.list.signed
#+end_example

The unsigned repository verifier checks:

- =Packages= contains =Package: elpa-nelix=
- package version and architecture rows exist
- =Filename= points at the copied =.deb=
- =Packages.gz= decompresses byte-for-byte to =Packages=
- =Release= contains hash metadata
- =sources.list= points at the generated suite/component

The signed repository gate creates a temporary GPG key, signs the
repository, exports =nelix-archive-keyring.gpg=, writes
=sources.list.signed=, and verifies both detached and inline
signatures with =gpgv=.

The static publication gate copies only the public repository payload
to =../nelix-apt-public=:

#+begin_example
../nelix-apt-public/
  pool/main/n/nelix/elpa-nelix_0.1.0-4_all.deb
  dists/unstable/Release
  dists/unstable/InRelease
  dists/unstable/Release.gpg
  dists/unstable/main/binary-all/Packages
  dists/unstable/main/binary-all/Packages.gz
  nelix-archive-keyring.gpg
  sources.list.signed
  sources.list.https
#+end_example

It refuses to publish a tree containing temporary GPG homes or private
key material.  The HTTP smoke serves that public tree from
=127.0.0.1=, runs =apt-get update= against an isolated temporary APT
state directory, downloads =elpa-nelix= through HTTP, and verifies the
downloaded package metadata with =dpkg-deb=.

=make apt-public-url-smoke APT_PUBLIC_URL=https://...= performs the
same rootless isolated APT update/download check against a final
published HTTPS URL.  The smoke downloads
=nelix-archive-keyring.gpg= from the repository root unless
=NELIX_APT_PUBLIC_KEYRING= points at a local keyring file.  The smoke also
checks that the downloaded =elpa-nelix= payload matches =DEB_VERSION=, so a
stale public repository does not pass.

=make apt-publication-preflight APT_PUBLIC_URL=https://...= is the
pre-upload static tree verifier.  It refuses the default
=example.invalid= URL, verifies the signed repository tree, checks that
=sources.list.https= exactly matches the public URL/suite/component,
rejects temporary GPG or secret material, compares =Packages.gz= with
=Packages=, and confirms the package payload referenced by =Packages=
exists.  It also checks that both the =Packages= row and the payload version
match =DEB_VERSION=.

=make deb-source-gate= builds the first-upload source artifacts with
=dpkg-buildpackage -S -sa= and lints =../nelix_0.1.0-4_source.changes=.
=debian/source/options= keeps local generated caches and repository
outputs such as =.cache/=, =nelix-apt-public/=, and =nelix-rpmbuild/=
out of the source diff, so developer smoke artifacts do not break the
source upload gate.

* Fedora / dnf workflow

install smoke implemented through a Fedora container gate.  The current
container proof uses =fedora:latest=, installs the RPM toolchain, builds
=nelix= and =emacs-nelix=, runs =rpmlint=, creates a local dnf repository,

* Release invariant

Distribution packages are only the bootstrap layer.  They must install
enough Nelix runtime to run:

#+begin_src shell
nelix --help
emacs -Q --batch --eval "(require 'nelix)" --eval "(require 'nelix-dsl)"
#+end_src

After bootstrap, package state should be driven by a Nelix manifest and
lock file rather than by hand-installed per-machine state.
