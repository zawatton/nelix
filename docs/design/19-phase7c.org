* Goal

Extend the DSL build-system vocabulary beyond
=stdenv / rust / python / go / emacs-package= with two new types:

- =node=    → =pkgs.buildNpmPackage= (requires =:npm-deps-hash=)
- =haskell= → =pkgs.haskellPackages.mkDerivation= (no required args)

* Changes (anvil-pkg-dsl.el)

- =anvil-pkg--known-build-systems= gains =node= and =haskell=.
- =anvil-pkg--validate-build-system=: =node= requires =:npm-deps-hash=
  (mirrors the =rust= =:cargo-sha256= contract and calls
  =anvil-pkg--reject-non-emacs-package-args=); =haskell= falls through
  the default arm (no required args, still rejects emacs-package-only
  keys).
- =anvil-pkg-render-nix= dispatch gains =('node ...)= and
  =('haskell ...)= arms.
- =anvil-pkg--render-node= emits =pkgs.buildNpmPackage= with an
  =npmDepsHash= field, via the shared render-derivation / pre-bs /
  post-bs helpers, mirroring =anvil-pkg--render-rust=.
- =anvil-pkg--render-haskell= emits =pkgs.haskellPackages.mkDerivation=
  via the same shared helpers (it accepts pname / version / src like the
  other builders).
