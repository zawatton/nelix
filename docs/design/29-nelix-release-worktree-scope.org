Automated audit:

#+begin_src sh
make release-scope-audit
#+end_src

Staging preview:

#+begin_src sh
make release-scope-stage
make release-scope-status
tools/nelix-release-scope-stage.sh --dry-run B
#+end_src

Explicit staging for one group:

#+begin_src sh
make release-scope-stage-b
# Equivalent low-level form:
tools/nelix-release-scope-stage.sh --stage B
#+end_src

The =--stage= mode refuses to run if the index already contains staged
changes.  Commit or unstage the previous group first.  To intentionally add
another group into the same staged set, set
=NELIX_RELEASE_SCOPE_ALLOW_DIRTY_INDEX=1= for that command.

Commit execution checklist:

#+begin_src sh
git diff --cached --name-status
make release-scope-status
make release-scope-audit
make check
make deb-local-gate

make release-scope-stage-a
git diff --cached --name-status
git commit -m "Keep anvil-pkg compatibility while introducing Nelix"

make release-scope-stage-b
git diff --cached --name-status
git commit -m "Add Nelix manifest CLI and native store"

make release-scope-stage-c
git diff --cached --name-status
git commit -m "Document Nelix architecture and release scope"

make release-scope-stage-d
git diff --cached --name-status
git commit -m "Add Debian RPM and repository publication packaging"

make release-scope-audit
git status --short
#+end_src

Do not run the audit and staging helpers concurrently.  Both use =git add -n=
or =git add= and therefore contend for =.git/index.lock=.  If the staged
status does not match the intended commit group, reset only the index with
=git reset=, inspect the worktree, and rerun the relevant =--stage= command.
The =--stage= helper also refuses a dirty index by default so adjacent commit
groups are not merged accidentally.

Do not run source tarball gates concurrently with compile/test targets.
=make check= creates and removes =*.elc= files, while =make fedora-source-gate=
archives the live worktree; running them in parallel can make =tar= fail with
=file changed as we read it=.  Run =make clean= before source-package gates.

This target prints the exact =git add -n= staging groups, verifies ignored
build output patterns, checks executable bits on CLI/verifier scripts, and
fails when any modified/deleted/untracked non-ignored path is not classified
into the release scope.  It also verifies Debian revision consistency across
=Makefile=, =debian/changelog=, Debian verifier defaults, and current release
documentation, and checks that the Debian autopkgtest is not marked
=superficial= while still invoking the installed CLI gate.

The staging helper uses the same A/B/C/D grouping as this document and defaults
to dry-run.  It stages files only when =--stage= is passed explicitly.
