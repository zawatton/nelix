SHIPPED 2026-06-13.  Single-verb scope on the current branch.

Phase 4 added profile inspection and rollback, but the public =pkg-=
surface still lacked an explicit "upgrade what is already installed"
verb.  Phase 6-B adds that gap-filling wrapper around =nix profile
upgrade= without changing the profile isolation model, error
contracts, or MCP naming rules established earlier in the project.

* Goal

| Before Phase 6-B                          | After Phase 6-B                                      |
|-------------------------------------------+------------------------------------------------------|
| Users can install, list, search, rollback | Users can also upgrade all or one installed element  |

| Upgrade semantics are implicit in raw Nix | Upgrade semantics are available through public Elisp |

* Locked decisions

** L61 — use the =".*"= matcher for upgrade-all, not =--all=

When =NAME= is nil, =pkg-upgrade= passes the two-character regex
=.*= to =nix profile upgrade=.  This upgrades every installed profile
element on Nix 2.18+ and stays portable across the project's declared
minimum supported versions.

Nix also supports =--all= as a newer alias, but that spelling is less
portable across the older 2.x range this repository still targets.
Using the documented matcher form avoids a version gate while keeping
the CLI invocation simple and explicit.

** L62 — blank strings are only treated as "all" on the MCP side

The public Elisp API accepts =nil= for "upgrade everything" and a
string or symbol for "upgrade one element".  Blank strings are
therefore treated as argument errors at the Elisp layer so typos do
not silently widen scope.

The MCP wrapper is more permissive because tool inputs arrive as
strings from agents.  There, =nil=, the empty string, and
whitespace-only strings normalize to "upgrade everything".

** L63 — post-upgrade side effects mirror =pkg-rollback=

After a successful =nix profile upgrade=, the implementation runs:

1. =(condition-case _ (pkg-list-generations) (error nil))=
2. =(anvil-pkg--rollback-replay-emacs-hooks)=

This exactly matches the whole-profile rollback path.  The generation
mirror is refreshed on a best-effort basis, and emacs-package
post-install hooks are replayed even if the refresh fails so
=load-path= stays aligned with the active profile contents.
