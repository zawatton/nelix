* Locked decisions

** L27 — MELPA upstream recipe fetch (opt-in)

- Public API: =anvil-pkg-emacs-fetch-melpa-recipe PNAME= → STRING |
  nil.  Returns the verbatim recipe body (trimmed) or nil on miss /
  error / when fetch is disabled.
- New defcustom =anvil-pkg-emacs-melpa-upstream-fetch= (boolean,
  default *nil*).  Opt-in to keep Phase 4-D semantics intact and avoid
  changing derivation hashes for existing users.
- Cache namespace =anvil-pkg:melpa-recipe= via =anvil-pkg-state=, TTL
  7 days.  Value plist =(:status SYM :recipe STRING-or-nil)= where
  status ∈ =(:hit :miss :error)=.
- HTTP layer reuses =anvil-pkg-compat-http-get= (text, not binary;
  recipes are short Elisp).  Timeout =anvil-pkg-emacs-http-timeout=
  (existing, 5 s default).
- Wire-in (=anvil-pkg--render-melpa-post-unpack=, dsl.el):
  - explicit =:melpa-recipe= → wins (existing).
  - =:melpa-synth 'never= → return nil (existing).
  - =:melpa-synth 'force= → synth, skip fetch (existing semantics
    preserved — =force= now explicitly means "ignore upstream").
  - =:melpa-synth 'auto= (default) AND fetch enabled → try upstream
    via fluid =anvil-pkg-emacs--render-fetch-fn=; on hit emit
    upstream recipe verbatim; on miss / error fall back to synth.
  - =:melpa-synth 'auto= AND fetch disabled (defcustom nil) → synth
    (current Phase 4-D behaviour).
- Render dispatch fluid: =anvil-pkg-emacs--render-fetch-fn= (=defvar=
  holding a 1-arg lambda PNAME → STRING-or-nil).  Tests rebind to a
  stub via =cl-letf= or =let=.  Default =funcall='s the public fetcher.

- Constant =anvil-pkg-emacs--default-melpa-files= holding a copy of
  =package-build-default-files-spec=:

- =anvil-pkg--synth-melpa-recipe= already does =(format "%S")= per
  element so nested =(:exclude ...)= sublists serialise correctly
  without further changes.
- User-supplied =:melpa-files= still wins; the default only applies
  when the keyword is omitted.
- Validator stays unchanged.  =:exclude= sublists are well-formed
  Elisp — only top-level strings are required when user supplies the
  list explicitly (validator already enforces).

* Open questions / non-goals

- *OQ20.* Should =force= attempt upstream fetch?  Decision: *no*.
  =force= is the user's explicit "synth, do not consult upstream"
  signal; =auto= is the discoverable default.
- *OQ21.* Should fetch be on by default?  Decision: *no, opt-in*.
  Adds latency to first install per package, changes derivation
  hashes vs Phase 4-D.  Defaulting nil keeps Phase 4-D behaviour and
  lets motivated users flip the switch.
- *Non-goal.* Detecting recipes in a *user-supplied* MELPA mirror /
  fork.  Always queries upstream =melpa/melpa@master=.
- *Non-goal.* Glob auto-detection from the actual source tree (would
  require unpacking the fetched tarball / cloning the git repo just
  to enumerate files).  L28 stops at "use a better default constant".
